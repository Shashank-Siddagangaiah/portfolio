Context

Complex Multi-Source SQL Was Creating Real Bottlenecks

  • Multi-source joins across EDW, AWM, and BIM required manually tracing inconsistent schemas and undocumented key relationships
  • Deduplication errors in ROW_NUMBER() patterns were difficult to isolate without a systematic, reproducible review process
  • Tableau SQL constraints prohibit CTEs inside Custom SQL — every report required complex nested subquery rewrites from scratch
  • Schema exploration consumed significant time each cycle: hunting table relationships across unfamiliar source systems
  • No knowledge retention — each report cycle restarted from zero with no reusable pattern library or documented decisions

Case Study

EDW Rewrite: Four-Section Query Delivered <2% Variance Across All Sections

  • Goal: Replace Online Accounts BIM dependency with validated AWM + EDW views
  • Scope: Four-section UNION ALL (Activation, Initiation, Inforce, DriverOnly), each independently validated against BIM
  • AI role: Join path analysis, ROW_NUMBER dedup review, Employee/Supervisor dimension mapping via USER_EVENT_DETAIL and community_agent
  • Architecture gap: DriverOnly ~43% lower than BIM — AI assisted root-cause analysis; documented as accepted architecture difference
Validated Outcomes
Activation section: <1% delta vs BIM
Initiation section: <2% delta (2021+ data)
Inforce section: <1.5% delta
Employee & Supervisor dimensions fully mapped
BIM dependency eliminated — EDW is now primary
Case Study

Paperless Report: Multi-Source Pipeline Designed and Validated With AI

  • Goal: Build reporting pipeline tracking paperless adoption across all insurance product lines
  • Sources: EDW, AWM, BIM/Eloqua — requiring cross-source CIF join validation and row-level deduplication
  • AI role: Pipeline design, CIF join path analysis, construction of missing mail and email validation queries
  • Constraint: All Tableau queries must use nested subqueries — AI structured every query to comply with the no-CTE rule
Validated Outcomes
Paperless indicators accurate across all product lines
CIF join validation resolved and documented
Missing mail and email validation queries built and verified
Tableau dashboards live tracking adoption trends
Issue resolution documented for team knowledge base
Impact

AI Has Fundamentally Changed How I Approach Engineering Problems

Before AI
Trial-and-error debugging — run query, observe error, guess at fix, repeat
Manual schema lookup — open docs, trace join keys by hand, repeat per report
Zero retained context — each report cycle restarted fresh with no pattern library
Ad-hoc SQL writing — requirements went directly to code with no design step
Isolated knowledge — solutions not documented; gaps repeated across cycles
After AI
Root cause first — systematic diagnosis before writing any fix
AI-assisted navigation — join paths explored and validated before building queries
Persistent knowledge base — validated patterns stored in Obsidian vault for reuse
Design before build — requirements fully explored and structured before SQL is written
Documented decisions — every architectural gap and resolution captured and auditable
Governance

All AI Output Is Reviewed and Validated — Accuracy Always Comes First

  • ✓
    Every AI-generated SQL is manually reviewed before execution against any production data source
  • ✓
    Cross-source validation queries reconcile counts between EDW, AWM, and BIM to confirm accuracy at every milestone
  • ✓
    No sensitive customer data is shared with external AI tools — queries use anonymized schema references and structural descriptions only
  • ✓
    Persistent knowledge base (Obsidian vault) retains validated patterns, known gaps, and architectural decisions for full auditability
  • ✓
    AI is a reasoning aid, not an authority — all output is one input to engineering judgment, never a replacement for it
